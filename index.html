4 Design Decisions That Defined This System

01

Schema-First Ingestion

Confluent Schema Registry with Avro enforcement at publish time, not consume time. Prevented 3 downstream outages caused by upstream schema drift. Any invalid event hits a dead-letter queue with automatic Slack alerting before it can corrupt warehouse state.

02

Serverless-First Compute

EMR Serverless over long-running clusters. Each Spark job provisions, runs, and tears down independently. Idle compute drops to zero between runs. This single change delivered 60% reduction in cluster idle time and eliminated the 2am "cluster stuck" class of incidents.

03

Incremental dbt With SCD Type 2

Full refreshes replaced with incremental strategies and Slowly Changing Dimension Type 2 on customer dimension tables. Transformation time cut by 75%. History preserved for auditability — critical for fintech compliance requirements.

04

Tiered SLA Alerting

DataDog monitors with P1/P2/P3 severity tiers mapped to business impact. P1 pages oncall within 2 minutes. P2 Slack channel. P3 next-business-day ticket. Result: incident response time dropped from 4 hours to 45 minutes average.

Production Incident

3am Schema Drift — What Actually Happened

Upstream team added a required Kafka field without coordinating. Spark job failed silently on deserialization. No alert fired. By morning, 6 hours of transaction data had missed the warehouse load window.

Root cause: No schema registry enforcement — consumers accepted any payload shape
Immediate fix: Dead-letter queue with deserialization error alerting deployed same day
Permanent fix: Schema Registry enforced at producer level — breaking changes now blocked at publish, not discovered at 3am
